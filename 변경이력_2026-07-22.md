## 1. Monitoring(관제) 화면 — 구조 재배치

- **상단 소개 문구 삭제**: 화면 최상단의 "병원 전체의 실시간 상태를 한눈에 관제합니다" 설명 문구 + "Monitoring" 큰 제목을 제거하고, 그만큼 확보된 세로 공간을 배치도 쪽으로 흡수.
- **시스템 상태 카드 → 설정으로 이동**: 원래 Monitoring 화면 최상단에 있던 시스템 상태 카드(FastAPI/Radar/Database/Security 연결 상태)를 **설정 > 일반 탭 최하단**으로 옮김. Monitoring 화면은 배치도·알람 확인에만 집중하고, 시스템 점검은 설정에서 하도록 역할을 분리.
- **층 선택 + 필터 + 검색창 한 줄 통합**: 예전에는 층 선택 버튼이 세로 칼럼을 차지하고, 상태 필터가 버튼 목록으로 한 줄 전체를 차지했음. 지금은 `1F 2F 3F … [검색창] [필터 아이콘]` 형태로 한 줄에 합쳐, 확보된 폭만큼 배치도가 실제로 넓어지도록 그리드를 재구성.
  - 필터는 더 이상 버튼 목록이 아니라 **필터 아이콘 클릭 → 체크박스 팝오버**(정상/움직임 없음/호흡 이상/낙상/센서 오류, 다중 선택 가능)로 변경 (`RoomFilter.jsx`).
- **하단 패널 우측 세로 → 배치도 하단 가로 배치**: 환자정보/알람/이벤트 로그 3개 패널이 배치도 오른쪽에 세로로 길게 붙어있던 것을 배치도 **아래쪽에 가로로 나란히** 배치. 화면 전체가 스크롤 없이 한 화면(뷰포트)에 고정되도록 레이아웃 자체를 재구성 (`.content--fixed`, `.app{height:100vh}`).
- **배치도 영역 단계적 확대**: 배치도와 하단 패널의 세로 비율을 요청마다 3:1 → 4:1 → 5:1로 확대 조정해, 배치도가 화면에서 차지하는 비중을 계속 키움.

## 2. Floor Editor(평면도 편집기) — 그리기 도구 & 데이터 구조

- **그리기 도구 확장**: 툴바에 사각형/다각형/벽/사각형벽/문에 이어 **계단, 엘리베이터** 버튼 추가.
- **계단/엘리베이터 = 문(door)과 같은 "순수 구조 심볼"로 재설계** (가장 큰 구조 변경):
  - 처음엔 "사각형 방을 그리고 유형을 계단/엘리베이터로 지정"하는 방식(방 데이터 모델 재사용)으로 구현했으나, "문처럼 심볼 자체를 바로 놓게 해달라"는 피드백에 따라 전면 재설계.
  - `rooms`/`walls`/`doors`와 나란한 **신규 `structures` 배열**을 데이터 모델에 추가(`ROOM_TYPES`에서는 완전히 제외). 클릭+드래그로 사각형을 그리듯 바로 놓이며, zones/beds/sensors/status 같은 방 데이터를 전혀 갖지 않는 순수 심볼.
  - 배경 없이 아이콘만 표시(라이트: 회색/검정 심볼, 다크: 배경색에 맞춰 자동 대비), 선택 시에만 파란 테두리로 강조. 테두리는 평소엔 없음.
  - 계단 아이콘은 한 차례 디자인(사선 해칭 패턴)을 새로 시도했다가, "예전 게 낫다"는 피드백으로 **기존 디딤판+화살표 디자인으로 롤백**.
- **방 생성 박스 위치 고정**: 캔버스를 클릭+드래그해서 방을 만들 때 뜨는 미리보기 박스가 좌측 신규 방 목록 때문에 아래로 밀려 보이던 문제를 고정 위치로 수정.
- **전체 구성도 미리보기 위치 이동**: 캔버스 위에 겹쳐 있던 미니맵을 Room Property(방 속성) 패널 **맨 아래**로 옮김(스크롤하면 보임). 가로 폭 100%, 세로는 비율(1000:700) 유지한 채 자동 계산. 지금 선택 중인 방은 초록색으로 강조 표시.
- **전체 배치 일괄 확대/축소**: "전체 선택" 토글을 켜면 배치도 전체(방+벽+문+구조물)를 감싸는 바운딩 박스와 모서리 리사이즈 핸들이 나타나, 드래그 한 번으로 전체를 비율 유지한 채 확대/축소 가능 (`GroupResizeHandles.jsx`).
- **호실 번호 여러 줄 입력 지원**: 호실 번호 입력창을 `<input>`에서 `<textarea>`로 바꿔 Enter로 줄바꿈 가능, 라벨도 `\n` 기준으로 여러 줄 렌더링.
- **방 이름 자동 접미사("호") 제거**: 이름 입력 시 자동으로 붙던 "호" 접미사를 제거(예: "응급의료센터"에 "응급의료센터호"로 붙던 문제 수정).

## 3. Floor Editor — 정렬 안내선(스마트 가이드, 캔바/미리캔버스 스타일)

- **이동 중 위치 정렬 안내선**: 방/구조물을 드래그로 옮기는 동안, 다른 방/구조물과 중심 또는 좌우상하 경계가 가까워지면 회색(라이트)/흰색(다크) 점선 안내선이 뜸. 위치를 억지로 붙이지는 않고 **시각적 안내만** 제공(처음엔 실제로 위치를 스냅시켰다가 "너무 꽉 잡는다"는 피드백으로 순수 안내선 방식으로 변경).
- **변(edge) 드래그로 한쪽 면만 리사이즈**: 기존에는 꼭짓점을 잡고 늘리면 가로/세로가 함께 바뀌었는데, 이제 **변의 중점 핸들**을 잡고 드래그하면 그 변(폭 또는 높이)만 늘어나거나 줄어들고 반대쪽 변은 고정됨.
- **크기 일치 안내선**: 변을 드래그해서 리사이즈하는 동안, 조정 중인 폭/높이가 옆의 다른 방·구조물과 같아지는 순간에도 안내선이 표시됨 — 크기를 눈대중이 아니라 정확히 맞출 수 있음.
- 이동 정렬 안내선과 크기 일치 안내선 모두 **방(rooms) ↔ 구조물(structures)** 서로를 비교 대상에 포함(방을 구조물에 맞추거나 그 반대도 가능).
- ※ "글꼴 크기를 개별/일괄로 조절하는 기능"도 이번 세션에 한 차례 추가했다가, 사용자 요청으로 **전면 롤백**(개별 설정 UI, 툴바 일괄 슬라이더 모두 제거) — 현재는 기존과 동일하게 박스 크기 기반 자동 폰트 크기만 적용됨.

## 4. 버그 수정

- **문(door) 배치 좌표 부정확 문제**: 마우스로 클릭한 지점이 아니라 옆으로 어긋난 위치에 문이 찍히던 문제. `getScreenCTM().inverse()` + `DOMPoint.matrixTransform()` 기반의 정확한 좌표 변환으로 통일(줌/팬 상태와 무관하게 정확).
- **호실 번호/센서 개수 입력 안 먹던 문제**: Room Property 패널이 선택 시점의 스냅샷(`selectedRoom`)을 그대로 참조해, 입력해도 `rooms` 배열은 갱신되지만 화면엔 반영 안 되던 버그 → 매번 `rooms`에서 최신 값을 다시 찾아 넘기도록 수정.
- **벽 그리기 더블클릭 시 점이 하나 더 찍히던 문제**: 더블클릭의 두 번째 클릭이 `mousedown`으로도 겹쳐 인식되던 것을 감지해 무시하도록 수정.
- **`.app` 높이 설정으로 인한 전역 스크롤 문제**: `min-height:100vh`였던 것을 `height:100vh`로 바꿔, 내부 콘텐츠가 커져도 문서 전체가 스크롤되지 않고 지정된 영역 내부에서만 스크롤되도록 수정(Monitoring 고정 화면의 근본 원인).
