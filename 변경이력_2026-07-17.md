# 변경 이력 — 2026-07-17

> 대상: `frontend/` React 코드베이스 (`smart-hospital-dashboard`)
> 작업 목적: 미구현된 탭 개발 진행 + 화면 기획 피드백 반영

---

## 1. 구현 방식 관련 참고사항

실제 `frontend/` 코드베이스는 `프론트엔드_디자인_기능_명세서.md` 초안의 "카드형 방 목록" 구조 대신, **SVG 폴리곤 기반 평면도 편집기**(층별 이미지 위에 병실을 다각형으로 그려 배치)로 이미 상당 부분 구현되어 있었음이 이번 점검에서 확인됨. 이번 작업은 새 구조를 만드는 대신 **기존 폴리곤 편집기 구조를 기준으로 미완성/오류 부분을 수정·완성**하는 방향으로 진행함.

## 2. 오류 해결 및 세부 고도화 진행

- **치명적 버그**: `RoomEditor.jsx`가 자기 자신을 import하여 `/editor/:floorId` 진입 시 무한 재귀로 크래시하던 문제 수정 (실제 동작하는 병실 목록/검색 사이드바로 교체)
- **데이터 단절 해소**: 병원관리 화면(층 추가/삭제, 평면도 업로드)과 층 편집기(방 그리기)에서 편집한 내용이 실시간 모니터링 대시보드에 전혀 반영되지 않던 문제 → 공용 상태 저장소(zustand, `localStorage` 영속화) 도입으로 모든 화면이 동일한 데이터를 공유하도록 수정
- **호실별 관리(1인실 기준)**: 병실을 "병상" + "화장실" 두 개의 센서 존(zone)으로 분리 관리. 신규 `/rooms` 페이지에서 호실 단위로 존 상태·환자정보·특이사항을 확인/수정 가능 (피드백 반영: "호실별 관리 필요할 것 같다")
- **알람 위치 상세화 + 정상해제/오탐 분류**: 알람 발생 시 "몇 호 화장실/병상"인지 위치를 명시하고, 해제 시 "정상 해제"와 "오탐(거짓 알람)"을 구분해 이력에 남기도록 변경. 신규 `/alarms` 페이지에서 활성 알람 처리 및 전체 이력 조회 가능 (피드백 반영: "알람은 실제로 클리어/거짓 알람 등으로 표현")
- **센서 커버리지 확장**: 병실 외 공용화장실/복도/공용공간/대기실/간호사스테이션에도 센서(zone)를 배치해 해당 구역에서도 낙상/호흡이상 이벤트가 발생할 수 있도록 시드 데이터 보강 (피드백 반영: "공용공간, 대기실, 화장실 등에도 센서 설치")
- **이벤트 시뮬레이션(F9)**: 실제 하드웨어 연결 없이 낙상/호흡이상을 임의 구역에 발생시키는 버추얼 트리거를 대시보드 상단 툴바 + 설정 화면(자동 반복 시뮬레이션)에 추가
- **다크/라이트 테마**: 기존 토글 기능은 정상 동작 확인. 저장된 선호가 없는 최초 접속 시에는 브라우저 언어(한/중/일 → 라이트, 그 외 → 다크)를 참고해 기본값을 정함 (피드백 반영: "아시아권 화이트 / 외국권 블랙 선호")
- **신규 페이지**: 호실별 관리(`/rooms`), 알람 관리(`/alarms`), 통계 보고서(`/stats`), 시스템 설정(`/settings`) 추가 및 사이드바 라우팅 연결(기존에는 사이드바 버튼에 클릭 동작이 전혀 없었음)
- **기타 수정**:
  - 층 평면도 업로드 필드명 불일치(`floor.image` vs `floor.floorMap.src`) 통일
  - 층 편집기 줌/그리드/스냅/실행취소·다시실행 버튼이 캔버스에 실제로 반영되지 않던 연결 오류 수정
  - SVG 내보내기 셀렉터 오류 수정
  - 검색창/전체화면/사이드바 접기 등 만들어져 있었지만 어디에도 연결되지 않았던 컴포넌트들을 실제 화면에 연결
  - Hospital Manager / Floor Editor 화면 전반에서 CSS 클래스가 정의되지 않아 스타일이 거의 적용되지 않던 문제 보강

## 4. 병실 고도화 및 관련 업데이트 진행

1차 작업 이후 추가로 전달된 8가지 정의/요구사항을 반영함. 주요 변경은 데이터 모델을 "1병실 = 1환자"에서 **"1병실 = 여러 병상(bed) 배열"** 구조로 재설계한 것이며, 이로 인해 병실/환자/알람 관련 컴포넌트 대부분을 다시 작성함.

- **다인실 지원**: `data/floorsData.js`에 `beds[]` 배열 기반 모델 도입. 1인실~8인실까지 동일한 코드 경로로 처리되며, 병상마다 독립된 센서 존(zone)과 환자(patient, 공석 가능)를 가짐. 시드 데이터에 4인실(205호)/3인실(304호)/8인실(310호) 데모 포함.
- **간호사 스테이션 삭제**: `ROOM_TYPES`에서 완전히 제거 (요청 반영: "간호사 스테이션은 필요 없음").
- **호실별 관리 화면 재구성** (`/rooms`, `pages/RoomsManager.jsx`): 층별로 그룹핑해서 모든 병실/공용구역(로비/대기실/화장실/복도/처치실/공용공간)을 표시. 상단에 "고위험 환자 TOP 5" 랭킹 패널 추가, 병실 펼침 시 병상 목록도 누적 낙상×2+호흡이상 점수가 높은 환자가 위로 오도록 정렬. 환자별 과거 병력/현재 증상도 함께 표시. 시드 데이터에 가상 누적 낙상/호흡이상 횟수를 채워 랭킹 동작을 바로 확인 가능하도록 함.
- **환자 관리 화면 신규 분리** (`/patients`, `pages/PatientsManager.jsx`): 기존에는 "호실별 관리"와 같은 라우트(`/rooms`)를 공유했던 것을 별도 페이지/라우트로 분리. 대시보드보다 확대된 층별 배치도를 보여주며, 다인실은 병상 배치 순서대로 환자 이름을 세로로 모두 나열(8인실도 전부 표시). 병실 클릭 시 모달로 배정된 모든 환자의 전체 이력(누적 낙상/호흡이상, 과거 병력, 현재 증상)을 확인 가능.
- **알림 전역 고정**: "새로운 알림이 없습니다" 토스트/알람 알림을 대시보드 화면에서만 렌더링하던 것을 `MainLayout.jsx`로 이동해 모든 탭에서 우측 하단에 항상 고정 표시되도록 수정.
- **센서 이상 알람 + 3단 분류**: 알람 상태를 낙상(danger)/호흡이상(warning)/움직임없음(inactive)/센서오류(sensor) 4종으로 확장하고, 해제 시 "정상 해제"·"오탐"·"센서 이상 확인" 3가지로 분류해 이력에 남김. 알람 관리(`/alarms`) 화면에서 4종 시뮬레이션 버튼과 3단 처리 버튼, 이력 필터를 모두 제공.
- **색상 규칙 확정 및 전면 적용**: 정상(🟢)/움직임 없음(🔵)/호흡 이상(🟠)/낙상(🔴)/센서 오류(⚫) 5단계 상태 체계를 `STATUS_META` 단일 소스로 정의하고, 병실 폴리곤·범례·필터·알람 카드·이벤트 타임라인·통계 화면 등 색상을 사용하는 모든 화면에 동일하게 적용.
- **기본 테마 라이트 고정**: 이전에는 브라우저 언어(아시아권/외국권)로 다크/라이트 기본값을 추정했으나, 이번 요청에 따라 저장된 선호가 없을 때는 항상 라이트 모드로 시작하도록 단순화 (`themes/ThemeContext.jsx`).
- **기타 정리**:
  - `.room-status` CSS 규칙에 높이/표시(display)가 빠져 상태 점(dot)이 화면에 전혀 보이지 않던 기존 버그 수정
  - `RoomEditor.jsx`/`Dashboard.jsx`의 환자 검색 로직이 삭제된 `room.patient` 단일 필드를 참조하던 버그 수정 → 병상 배열(`room.beds[].patient`) 기준으로 재작성
  - 통계 보고서(`/stats`)를 5단계 상태 분포 및 센서 이상 해제 통계까지 표시하도록 확장
  - `.icon-button-sm`, `.bed-stepper`, `.alarm-resolve-btn.sensor`, `.alarm-card.sensor`, `.event-item.sensor` 등 이미 JSX에서 쓰이고 있었으나 스타일이 없던 클래스 보강

## 5. 메뉴 최종 정리(6개) + 기능 갭 보완 + 전체 디자인 개편

사용자가 정리한 최종 메뉴 구조(Monitoring / 호실별 관리 / 환자 관리 / 알람 관리 / 통계 / 설정, 총 6개)를 기준으로 기능 갭을 메우고, 디자인을 전면 개편함.

### 5.1 메뉴 구조 정리
- 예전에는 사이드바의 "대시보드"와 "모니터링"이 둘 다 같은 화면("/")을 가리켜 사실상 중복이었음. 최종 정리안에는 별도 "대시보드" 메뉴가 없고, **"Monitoring" 하나가 KPI 5종 + 층별/호실별 실시간 상태 + 이벤트 로그 + 환자 상세 정보 + 실시간 위험 알림을 모두 포함**하는 것이 맞는 구조로 확인됨. 직전 라운드에서 만들었던 KPI 전용 `/monitoring` 페이지를 다시 `pages/Dashboard.jsx`(라우트 `/`)에 합치고, 사이드바를 정확히 6개 메뉴로 정리함. KPI는 `components/dashboard/MonitoringKpiPanel.jsx`로 분리해 재사용.
  - **Monitoring KPI 5종 확정**: 현재 관제 중 환자 수 / 평균 호흡수 / 센서 연결률 / 오늘 위험 발생 건수(전체) / Critical 이벤트 수(그 중 낙상=danger 등급만 별도 집계). `utils/stats.js`의 `getMonitoringStats`가 계산.
  - 삭제 도구가 없어 더 이상 쓰이지 않는 `pages/MonitoringPage.jsx`, `components/dashboard/KpiPanel.jsx`는 상단에 DEPRECATED 주석만 남기고 라우팅/임포트에서는 완전히 제거함.
- **병원/층 구조 관리 접근성 문제 발견 및 수정**: `HospitalManager.jsx`(병원정보/로고/층 관리/병동 관리)와 `FloorEditor.jsx`(호실 생성·수정·삭제, 위치/크기 편집)는 이미 만들어져 있었지만 사이드바 어디에도 연결되지 않아 실제로는 접근이 불가능했던 것으로 확인됨(라우트 `/hospital`, `/editor/:floorId` 자체는 존재).
  - "병원 정보 관리"·"층 및 병동 설정"은 최종 스펙상 **설정(⚙)의 핵심 기능**이므로, `HospitalInfoCard`/`LogoUploader`/`FloorManager`/`BuildingManager`를 `SettingsPage.jsx`의 "병원 · 층/병동 관리" 탭 안으로 그대로 재사용해 통합함.
  - "호실별 관리" 화면에는 층별로 "구조 편집" 버튼과 상단에 "층 추가" 버튼을 추가해 `FloorEditor`로 바로 진입할 수 있게 함(호실 생성/수정/삭제, 위치·크기 편집, 센서 자동 등록).

### 5.2 환자 관리 — 병실 이동 + 최근 이벤트 이력
- store에 `movePatient(fromFloorId, fromRoomId, fromBedId, toFloorId, toRoomId, toBedId)` 액션 추가 — 다른 병상이 공석일 때만 환자를 이동시킴("호실 배정" 기능 보완).
- `PatientDetailModal`(환자 관리 화면의 병실 상세 모달)에 환자별 "이동" 버튼(다른 층/병실/병상 선택 드롭다운)과, `eventLog`를 병상의 zoneId로 필터링한 "최근 이벤트" 목록을 추가함.

### 5.3 알람 관리 — 확인(ACK) + 이력 검색 + 심각도 라벨
- store에 알람 엔트리 `acked` 필드와 `ackAlarm(alarmId)` 액션 추가 — "처리 완료(정상해제/오탐/센서이상)"와는 별개로 "확인만 함"을 표시할 수 있게 함.
- 활성 알람 표에 Critical/Warning/Info 심각도 배지 추가(기존 5단계 상태·색상 규칙은 그대로 두고 그 위에 심각도 라벨만 매핑: 낙상=Critical, 호흡이상=Warning, 움직임없음·센서오류=Info).
- 이벤트 이력에 호실/구역/환자명 검색창 추가.

### 5.4 통계 — 기간별 추이 + 호실별 빈도 + 호흡수 변화
- `utils/stats.js`에 `getEventTrend`(일/주/월 단위 위험 발생 추이), `getRoomFrequency`(호실별 위험 발생 빈도 순위), `getRespirationTrend`(평균 호흡수 변화 추이) 추가.
  - **참고**: 평균 호흡수 변화는 실제 활력징후 시계열 로그가 아직 없어(60GHz 레이더 미연결·버추얼 인터페이스 단계) 현재 평균값 기준 시드 기반 참고용 추이임. 실제 센서 이력이 연동되면 이 함수만 교체하면 되도록 분리해둠.
- 외부 차트 라이브러리 없이 순수 CSS로 그리는 `components/stats/TrendBarChart.jsx` 추가.
- 통계 화면에 센서 연결률/평균 호흡수 타일, 기간별(일/주/월 토글) 추이 그래프, 호실별 위험 발생 빈도 순위, 호흡수 변화 추이 섹션을 추가함(기존 상태 분포/알람 처리 통계/층별 현황은 유지).

### 5.5 설정 — 병원/층 관리 통합 + 사용자 및 권한 + 알림 소리/팝업
- `SettingsPage.jsx`를 탭 구조(일반/병원·층·병동 관리/사용자 및 권한/알림/데이터)로 재구성.

## 6. 일부 오류 및 중복 구현 사항 제거 조치

9가지 오류 및 중복 사항 제거 문제를 수정함.

### 6.1 알림 소리 — 사이렌 반복 패턴
- `components/common/NotificationToast.jsx`의 단발 비프음(`playBeep`, 0.35초 사인파 1회)을 `playSiren`으로 교체. 사이렌 특유의 톱니파(sawtooth)로 520Hz↔1050Hz를 오가며 0.55초 주기로 3회 반복(총 약 1.65초)해, 멀리 있는 사람도 상황을 인지할 수 있는 "웨엥-웨엥-웨엥" 패턴으로 변경.

### 6.2 알림 토스트 닫기 버튼 위치 고정
- `.notification-toast-stack .notification-toast`에 고정 폭(`width:400px`, `max-width:calc(100vw - 64px)`)을 지정하고, 라벨 텍스트에 `flex:1` + 말줄임(ellipsis), 닫기 버튼에 `flex-shrink:0`을 적용. 기존에는 카드마다 텍스트 길이에 따라 폭이 달라져 X 버튼 위치가 카드마다 제각각으로 보이던 문제를 해결.

### 6.3 알람 해제 2단계로 단순화 (정상 해제 / 오탐)
- 활성 알람을 처리할 때 "센서 이상 확인" 옵션을 없애고 "정상 해제"/"오탐" 2가지만 남김. `AlarmPanel.jsx`, `components/patient/PatientPanel.jsx`(호실 전체 해제), `pages/AlarmsManager.jsx`, `pages/RoomsManager.jsx`(구역 상세의 전체 해제)에서 센서 이상 버튼과 미사용 `Wrench` 아이콘 import를 제거.
- 존(zone) **상태**로서의 "센서 오류"(⚫, `STATUS_META.sensor`)와 알람 **유형**으로서의 "센서 오류" 자체는 기존 5단계 색상 규칙을 그대로 유지하기 위해 남겨둠 — 이번 요청은 "해제 시 분류 액션"만 2단계로 줄이는 것으로 해석함. 과거에 "센서 이상 확인"으로 처리된 이력(`eventLog`)의 표시(통계/이벤트 타임라인/알람 이력 필터)는 그대로 유지.

### 6.4 환자 관리 화면 레이아웃/배치도 개선

- 방 폴리곤 위에 상태색이 두 번(폴리곤 채우기 + 붉은 점멸 레이더 아이콘) 표시되던 문제를 해결: `PatientFloorCanvas.jsx`에서 `SensorOverlay` 렌더링을 제거하고, `PatientRoomLabel.jsx`에 테두리 없는 작은 점 하나만 호실명 앞에 추가해 상태를 보조 설명하도록 재작성.
- 다인실 환자 이름을 좌/우 2열로 나눠 표시(`PatientRoomLabel.jsx`) — 예: 8인실은 좌측 4명 / 우측 4명. `PatientRoomPolygon.jsx`가 방의 실제 폭/높이(px)를 계산해 넘겨주면, 라벨이 열 폭·줄 간격에 맞춰 글자 크기를 자동 조절해 글자가 밀리거나 잘리지 않도록 함.
- 환자 이름 마스킹 제거: `data/floorsData.js` 시드 데이터의 "이OO" 형태 마스킹 이름을 전부 실제 이름 형태(예: "이순자", "박옥희")로 교체(병원 관계자만 열람하는 화면이므로).
- 배치도가 주어진 영역을 최대한 활용하도록 `<svg preserveAspectRatio="none">`으로 변경(대시보드·환자관리 캔버스 공통) — 박스와 SVG viewBox 비율이 달라 생기던 레터박스(빈 여백)를 없앰.

## 7. 층별 호실 영역 출력 영역 수정

`preserveAspectRatio="none"`가 실제로는 "글자가 늘어져 보이는" 새로운 오류를 잡기 위함

### 7.1 상태 표기에서 이모지 제거 → 단색 원(dot)으로 통일
- Monitoring 우측 상단 범례(`FloorLegend.jsx`), 층별 필터(`RoomFilter.jsx`), 통계 보고서의 "상태 분포"·"층별 현황"(`StatsReport.jsx`)에서 🟢🔵🟠🔴⚫ 이모지 표기를 제거하고, 이미 정의돼 있던 테두리 없는 단색 원 `.legend` 클래스로 통일.
- "일관되게 모든 부분을 변경해달라"는 요청에 따라 위 3곳뿐 아니라 `RoomsManager`/`AlarmsManager`/`AlarmPanel`/`PatientPanel`/`PatientDetailModal`/`RoomEditor`/`RoomProperty`(병실 유형 드롭다운은 HTML `<option>` 특성상 아이콘을 넣을 수 없어 텍스트만 유지) 등 `STATUS_META.emoji`를 쓰던 모든 곳을 함께 정리함.
- `PatientPanel.jsx`의 구역 상태 행에 인라인 스타일 점 + 이모지가 중복 표시되던 것도 이번에 함께 정리해 점 하나만 남김.

### 7.2 모니터링 날짜를 실제 달력으로 선택 가능하도록 변경
- `TopToolbar.jsx`의 `.today-box`가 `new Date().toLocaleDateString()`만 정적으로 표시하던 것을, 숨겨진 `<input type="date">` + `showPicker()`(미지원 브라우저는 focus로 대체)를 연결해 클릭하면 실제 브라우저 캘린더가 뜨고 연/월/일을 직접 선택할 수 있도록 변경.

### 7.3 "이벤트 시뮬레이션" 명칭 변경 + 존재 이유 설명
- `TopToolbar.jsx`의 "이벤트 시뮬레이션" 드롭다운 버튼/옵션에서 "시뮬레이션"이라는 단어를 모두 빼고 "테스트 이벤트 발생"/"낙상 테스트" 등으로 이름을 바꿈.
- **왜 남겨두었는가**: 이 기능은 실제 낙상이나 호흡 이상을 기다리지 않고도 알림음·팝업·사이렌 등 알림 파이프라인이 정상 동작하는지 즉시 확인할 수 있게 해준다. 소방 점검에서 화재경보를 눌러 테스트하는 것과 같은 목적으로, 병원에 납품된 뒤에도 신규 직원 교육이나 정기 점검 때 유용하게 쓰일 수 있어 완전히 삭제하지 않고 이름만 바꿔 남겨두었다. 불필요하다고 판단되면 언제든 삭제 가능.

### 7.4 모니터링 배치도 영역 확대 + 호버 팝업 정상 출력
- **레이아웃 자체 재구성**: 층 선택 버튼이 세로 80px 칼럼을 차지하던 것을 `variant="horizontal"`로 캔버스 위쪽으로 옮기고, `.dashboard` 그리드를 3단(80px+배치도+340px)에서 2단(배치도+340px)으로 줄여 그만큼 배치도 폭을 실제로 넓힘 — 이전 라운드의 "비율만 확대"가 아니라 화면 구조 자체를 바꾼 것.
- **글자가 늘어져 보이는 문제 원인 규명 및 원복**: 지난 라운드에 배치도의 빈 여백(레터박스)을 없애려고 `<svg preserveAspectRatio="none">`을 적용했는데, 이는 SVG 좌표계의 가로/세로 스케일이 박스 비율에 따라 서로 달라져 방 안의 글자와 도형이 비율에 안 맞게 눌리거나 늘어나 보이는 부작용이 있었다. 왜곡 없는 균일 스케일(`meet`, 기본값)로 되돌리고, 대신 위 레이아웃 재구성으로 박스 자체를 넓혀 빈 여백을 최소화하는 방향으로 접근을 바꿈.
- **호실 호버 팝업이 깨지던 문제 수정**(`components/room/RoomTooltip.jsx`): 팝업 높이 계산식(`56 + 줄수*16`)이 실제 렌더링 높이(패딩+제목+부제+줄당 약 25px)보다 작게 잡혀서, 병상이 많은 방(예: 8인실 310호)은 아래쪽 내용이 잘려 보였음 → 줄당 높이를 넉넉하게 재계산. 또한 화면 오른쪽 끝에 가까운 방 위에서는 팝업이 SVG 좌표계(가로 1000) 밖으로 나가 오른쪽이 통째로 잘려 보였는데, 캔버스 경계를 벗어나면 반대쪽(왼쪽/위)으로 자동으로 뒤집어 표시하도록 수정.

### 7.5 Monitoring "환자 정보" 패널 재구성 (2차)
- 코드 연결 자체는 정상이었지만, 병실을 선택하지 않았을 때 보이는 빈 상태 화면의 클래스들(`.patient-empty`, `.patient-empty-left/right`, `.patient-empty-icon`, `.patient-line`)에 CSS가 **전혀 정의되어 있지 않아** 아이콘·제목·문구가 정렬 없이 그대로 쌓여 보이는 것이 실제 원인이었음을 이번에 코드 대조로 확인함. 좌/우로 나뉘어 있던 어정쩡한 구조를 아이콘 + 제목 + 안내문구가 세로로 중앙 정렬되는 단일 카드로 다시 만들고 스타일을 새로 추가함.

### 7.6 알람 패널 스크롤 추가
- `.alarm-panel`에 `overflow:hidden`만 있고 내부 목록(`.alarm-list`)에는 스크롤이 전혀 없어서, 그리드 행 높이(300px)를 넘는 알람은 스크롤도 안 되고 그냥 잘려서 보이지 않던 문제를 발견 → `.alarm-panel`을 세로 flex 컨테이너로, `.alarm-list`를 `flex:1; overflow-y:auto`로 바꿔 헤더는 고정하고 목록만 스크롤되도록 수정.
- 알람 개수 배지(`{alarms.length}`)와 처리 시 목록에서 사라지는 동작은 코드 확인 결과 이미 정상 동작하고 있었음(스토어의 `resolveAlarm`이 해당 알람을 `alarms` 배열에서 제거하고, 배지도 같은 배열 길이를 그대로 참조) — 스크롤이 없어 가려져 있던 항목들 때문에 마치 처리해도 반영이 안 되는 것처럼 보였을 가능성이 높음.

### 7.7 호실별 관리 필터: 유형별 → 층별
- `pages/RoomsManager.jsx`의 필터 칩을 병실 유형(전체/병실/공용화장실/…) 기준에서 층(전체/1F/2F/3F …) 기준으로 변경. 화면이 이미 층별로 그룹핑되어 보여지는 구조라, 층 필터가 실제 사용성에 더 맞다고 판단.

### 7.8 환자 관리 배치도 전체 영역 활용
- 환자 관리 화면 전용 카드(`.patient-floor-canvas`)에 `height:min(78vh,860px)`라는 자체 높이 제한이 있어 대시보드의 배치도와 다르게 동작했던 것을 발견. 페이지 전체를 세로 flex 레이아웃(`.patients-page{height:100%}`)으로 만들고 캔버스 영역이 `flex:1`로 남는 세로 공간을 모두 가져가도록 해, 대시보드의 배치도와 동일한 방식(그리드/플렉스 stretch)으로 통일함.

### 7.9 오탐 처리 → 센서 오류 누적 + 이벤트 이력의 "센서 이상" 필터 제거
- `resolveAlarm`에서 `resolution === "false_alarm"`(오탐)으로 처리될 때마다 해당 구역(zone)에 `sensorErrorCount`를 누적하도록 store 로직 추가. 오탐이 반복되는 구역은 실제로 레이더 센서 자체의 오작동일 가능성이 높다는 판단 하에, 알람을 개별적으로 "오탐 처리"한 이력과는 별개로 구역 단위 누적치를 추적함.
- `utils/stats.js`에 `getSensorErrorStats` 추가 — 누적치가 높은 구역 순으로 정렬. `StatsReport.jsx`에 "센서 점검 필요 구역 — 오탐 누적 기준" 패널로 노출, 기존에 항상 0이었던(이제 발생 자체가 불가능한) `resolvedSensor`/"센서 이상 확인" 통계 항목은 제거.
- `AlarmsManager.jsx` 이벤트 이력의 "센서 이상" 필터 칩(레거시, `resolution === "sensor_error"`)을 완전히 삭제. `data/floorsData.js`의 `RESOLUTION_LABEL`에서도 `sensor_error` 항목 제거.

### 7.10 설정 > 데이터 탭을 레이더 센서 연동 설정으로 교체
- 기존 "데모 데이터 초기화" 버튼(`resetDemoData` 액션 자체는 store에 남겨두되 화면에서는 제거)을 없애고, 실제 레이더 센서 게이트웨이 서버와 연동하기 위한 화면으로 교체함.
  - **서버 연동 설정**: 통신 방식(MQTT/WebSocket/REST), 서버 주소, 포트, API 키, 자동 재연결 여부. store에 `sensorIntegration` 상태 + `updateSensorIntegration` 액션 추가(로컬 저장·영속화).
  - **구역별 센서 기기 매핑**: 모든 층/구역을 표로 나열하고 각 구역에 실제 장비 고유 ID(시리얼 번호 등)를 입력하면 "연동됨" 배지로 표시. store에 `zone.deviceId` 필드 + `updateZoneDeviceId` 액션 추가.
  - "연결 테스트" 버튼은 아직 실제 하드웨어에 연결하지는 않으므로(버추얼 인터페이스 단계) 입력값 존재 여부만 확인하는 수준으로 동작하며, 실제 장비 연동 시 이 부분만 실제 핸드셰이크 요청으로 교체하면 됨.

### 7.11 3층에 샘플 평면도 이미지 추가
- `config/hospital.config.js`의 3F `floorMap.src`에 외벽/방 구획/문 표시/복도 해칭/방위·축척 표시가 있는 건축 도면 스타일 SVG를 데이터 URI로 인라인 삽입 — 실제 평면도 이미지를 업로드했을 때 배경에 어떻게 보이는지 미리 확인할 수 있음. 실제 운영 시에는 병원관리 > 평면도 업로드에서 실사진/스캔 이미지로 그대로 대체 가능.
