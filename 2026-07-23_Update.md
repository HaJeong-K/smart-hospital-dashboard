## 1. 모니터링(Monitoring) 화면 UI 재구성

- **레이아웃 한 줄로 병합**: 예전엔 "검색/층선택/필터"와 "테스트 이벤트/새로고침/확대"가 각각 다른 줄을 차지했는데, 이제 한 줄에 왼쪽(검색 → 층 선택 → 상태 필터) / 오른쪽(테스트 이벤트 발생 → 새로고침 → 확대)으로 나눠 담아 세로 공간을 절약했다.
- **상태 필터 동작 변경**: 필터로 선택하지 않은 상태의 방을 화면에서 지우는 대신, 회색으로 표시하고 선택한 상태의 방만 원래 상태색을 유지하도록 변경(배치도 전체 구조는 그대로 보임).
- **날짜 선택 패널 제거**: Monitoring은 "현재 시점"만 보여주는 게 맞다는 방향에 따라 날짜 선택기를 완전히 제거.
- **새로고침 버튼 아이콘만 표시**: 텍스트 라벨을 빼고 아이콘만 남김.
- **2개 층 동시 표시**: 선택한 층 하나만 보여줄 때 배치도 옆 공간이 많이 남는다는 피드백에 따라, 층이 2개 이상이면 배치도를 왼쪽/오른쪽 두 개 동시에 보여준다.
  - 왼쪽/오른쪽에 각각 어떤 층을 띄울지 상태 필터 옆 "층 선택" 팝오버에서 독립적으로 고를 수 있다(오른쪽을 직접 고르지 않으면 인접 층이 자동으로 채워짐).
- **"Patient Information" 패널 확대 버튼 동작 변경**: 클릭 시 `/patients` 탭으로 이동하던 것을, 같은 화면 위에 모달(호실 상세)이 뜨도록 변경 — 대시보드를 벗어나지 않음.

## 2. 기능 위치 이동

- **Report(CSV 다운로드) 버튼**: Monitoring 툴바 → **통계 보고서** 화면으로 이동(리포트 다운로드는 통계 화면의 기능이라는 판단).
- **과거 이력 조회**: Monitoring에서 뺀 "날짜 선택 → 과거 조회" 기능을 사이드바 신규 메뉴 **"과거 이력"**(`/logs`)으로 분리 이전.
  - 알람 관리 화면에 있던 "이벤트 이력" 테이블(발생/정상해제/오탐 필터 + 검색)을 그대로 이관하고, **기간(시작일~종료일) 조회 필드**를 새로 추가.
  - 알람 관리(`/alarms`)는 이제 "활성 알람 확인/처리"라는 실시간 대응 목적에만 집중.
- **평면도 업로드/제거**: Floor Editor(방 그리는 캔버스 화면)에서 완전히 빼고, **설정 > 병원·층·병동 관리**로 일원화(아래 3번 참고).

## 3. 설정 > 병원·층·병동 관리 — 편의 기능 추가

- 층 카드의 버튼 구성을 **업로드 → 평면도 제거 → 에디터 → 삭제** 순서로 정리.
  - **평면도 제거** 신규 추가: 배경 이미지만 지우고 이미 그려둔 방/벽/문/구조물은 유지. 평면도가 없으면 버튼 비활성화.
- **Floor Editor 상단에서 층 이름 직접 수정** 가능하도록 변경(예전엔 읽기 전용 텍스트 → 입력창으로 변경, 저장 없이 즉시 반영).
- Floor Editor 캔버스: **Ctrl+좌클릭 드래그로 화면 이동(pan)** 추가 — 확대된 상태에서 그리기 도구를 켠 채로도 바로 이동 가능.
- Floor Editor에 **에스컬레이터** 구조물 타입 신규 추가(계단/엘리베이터와 동일한 방식으로 드래그 생성).
- 문/계단/엘리베이터/에스컬레이터 심볼 아이콘 정리:
  - 계단·에스컬레이터는 사용자가 제공한 참고 이미지를 그대로 사용.
  - 문 색상을 벽/사각형 벽과 동일한 회색으로 통일.

## 4. 통계 보고서 화면 확장

기존(기간별 추이, 호실별 위험 발생 빈도, 상태 분포, 알람 처리 통계, 센서 오탐 구역, 층별 현황)에 아래 5개 항목을 추가:

1. **시간대별 발생 분포** — 심야/오전/오후/저녁 4구간별 이벤트 건수 + 낙상/호흡이상 세부 건수
2. **낙상 다발 구역 TOP** — 낙상만 따로 집계한 호실 순위
3. **호흡이상 다발 구역 TOP** — 호흡이상만 따로 집계한 호실 순위
4. **구역 유형별 발생 비교** — 병실/화장실/복도/공용공간 등 유형 단위 집계
5. **환자별 고위험군 요약 TOP 5** — 호실별 관리 화면과 동일 기준(낙상×2+호흡이상 점수)으로 통계 화면에도 요약 노출

모두 실제 `eventLog`/환자 데이터 기반으로 계산되며 하드코딩 없음.

## 5. 기타
